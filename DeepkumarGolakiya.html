There are a number of pipeline components you can access through the textplumber package. You will have an opportunity to learn about this in labs, but documentation is available here.

To speed up preprocessing some of the pipeline components store the preprocessed data in a cache to avoid recomputing them. Run this as is - it will create an SQLite file with the name of your dataset option in the directory of the notebook. This will speed up some repeated processing (e.g. tokenization with Spacy).

5. Project Report¶

5.1 Problem Statement¶

Problem Statement¶

This project uses machine learning to address the task of automatically classifying historical newspaper articles by the different genres in the dataset. The particular dataset is taken from the Papers Past archive which includes the categories of genres such as: Letter, Notice, Review, Fiction, Recipe, Poetry or verse, and Obituary. These particular categories of genre showcase what are the different writing styles, structure, and purposes for making a genre classification meaningful rather not making it a challenging task.

The guiding research question is as follows:

"How well can we classify historical newspaper articles by genre?"

So this is a multi-classification task in which each article actually belongs to one particular genre. For getting broad performance view, we can use overall accuracy, but I think it is not enough for this particular dataset because of genre imbalance. So by keeping this in mind we need to focus on macro-averaged precision, recall and F1-score, which will ensure us that genres which are underrepresented like Obituary or Poetry are evaluated and represented fairly.

This particular study explores:

  • How good the different linguistic features like lexical, syntactic or semantic differentiate the genres which are in our dataset.
  • Which genres are mostly confused and what are the reasons behind it.
  • How is the preprocessing and feature choices affect the overall classifier performance.

These goals are aligned with interests which are scholarly broader in genre hybridity and also the linguistic characteristics of its historical texts. By doing multiple experiments, this project explores which combinations of different features and cleaning types support the best for conducting genre separation, and also to find out where actually the traditional classifier actually struggles to identify the genre.

5.2 Dataset¶

5.2 Dataset¶

The dataset that is used in this assignment was taken from the Papers Past archive and it includes a collection of newspaper articles that are labeled by the genre. The dataset has a span of seven genres which are Letter, Notice, Review, Fiction, Recipe, Poetry or verse, and Obituary, and it is formatted in the JSON file where each of the entry contains the article text and then its genre label.

The dataset has a key characteristic of genre imbalance. Genres like Notice and Recipe are highly shown or represented in the dataset while genres like Obituary and Poetry are significantly lesser in terms of examples. These particular imbalances in the dataset have a major role in evaluation, metrics and F1-score.

There is a variation in stylistic and structural features among different genres in the dataseet. Genre like Notice have a concise and formulaic phrasing, whereas Letter and Review genres have a more personal touch tone and narrative language. This variations show the feature design choices and how it encourages to include both lexical and semantic representation to capture the diversity.

There is a preprocessing challenge which includes OCR noise and also the inconsistent punctuation, which gives the reason to use text cleaning steps such as whitespace normalization and character replacement.

Overall, the dataset was realistic and difficult enough that provide a good challenge for building the model to classify the historical texts by genre.

5.3 Model¶

Model¶

In this project, we will be using two classifiers in total: Logistic Regression and Decision Tree. Logistic regression is a very good linear baseline which has a good amount of interpretability and has a robust aspect to it on sparse data which is high-dimensional in nature such as token features. On the other hand, Decision Tree aims to catch interactions which are non-linear and provide a transparency on rule basis, which is very important and useful when we are using genres that are structured or layout driven.

To identify linguistic representation effect, we test six feature configurations across these 2 particular classifiers. Each of the configuration was different either in feature type, the preprocessing, or even both, by keeping the classifier constant. That helped us to isolate the importance and impact of each component.

Features Used¶

We basically used the following feature types that are taken from very different level of linguistic and structural information:

  • Token n-grams (unigrams, with optional bigrams): lexical surface-level features
  • Part-of-Speech (POS) tags: syntactic structure
  • Lexicon-based counts (using Empath): affective/topic categories
  • Text statistics: some structural features like line width, line spacing, and document length

All of the features were then implemented by using pipeline framework known as textplumber, which is combined with FeatureUnion for getting a modular composition that is flexible.

Preprocessing¶

For implementing preprocessing we used TextCleaner with two primary settings which are as follows:

  1. Whitespace normalization
  2. Character replacements (That helps in replacing OCR-specific curly quotes and the various artifacts)

To get new combinations, I made sure that each feature was combined or paired with a preprocessing method that is not previously executed or used with it.

Feature Selection¶

In one of the combinations we additionally used SelectKBest with mutual information for feature set by selecting the top 5 features. This helped us to know whether POS features’ statistical filtering helps us improve our classification performance, and also to know whether or not feature selection is a meaningful preprocessing step or not.

Each experiment was tracked and evaluated with every step to ensure that the reflection of result was only because of one variable change at a particular time. That is the best practice for testing and interpretability of the model.

5.5 Discussion¶

The results show that performance of the genre classification differentiates largely by both feature and the model architecture. For genre and evaluation metrics, Logistic Regression models clearly outperforms the Decision Tree. This particular finding supports the hypothesis that in historical newspaper text, surface-level lexical and semantic signals are more effective than syntactical or structural ones, especially where genres overlap linguistically and stylistically such as in a multiclass context.

While doing close examination on genre-specific performance, it illustrated that the Notice and Recipe were consistently the easiest ones to classify rather than to other genres. These easier-to-classify genres usually use formulaic and domain-specific vocabulary, which was captured well by token-baseed models that are Combinations 1 and 2, and it was even well maintained under weaker Decision Tree settings. For example, In Combination 4, which is the weakest model, Recipe still achieved a high recall, showing that some genres can be differentiated with minimal modelling power.

In contrast, Letter, Review, and Poetry or verse were not classified effectively across different models. These particular genres are mostly narrative and personal in nature, which then lessens the power of surface lexical features. Letter was confused with Notice at a higher rate, relects their shared use of formal, dated expressions and greetings, that is an issue that is highlighted in the confusion matrices in Combinations 1, 2, and 3.

The introduction of lexicon-based features was really effective for Review and Poetry in the Combination 3, where it improved the F1 score by capturing thesemantic and affective tone rather than raw frequency. This gives the suggestion that affect-rich or evaluative genres are the one that are benefitting more from the semantic abstractions (e.g., "anger", "trust", "business") than from syntactic structure.

The attempt that was made in Combinations 5 and 6 to improve syntactic modelling with POS features and feature selection did not result with meaningful improvement. Even also applying SelectKBest, the F1 scores for Letter, Review, and Recipe remained at 0.000, suggesting that POS features lacked the granularity in genre separation in historical text, or the Decision Trees are too shallow to leverage them effectively and precisely.

Limitations¶

In this study, one limitation is that there is a reliance on pre-defined genre labels, which may be not accountable for the hybrid texts or editorial ambiguity. Additionally, some of the models are trained on default settings that can result in limitting the performance of more complex models such as Decision Tree. There is also relatively small support for lower-performing genres (e.g., Obituary), that could be a source of variance in evaluation.

5.6 Conclusion¶

Conclusion¶

This particular project did an investigation on genre classification of historical newspaper by using a set of combination features such as lexical, syntactic and structural across two classifiers: Logistic Regression and decision Tree. By doing evaluation of six models, it showed that Logistic Regression, which was paired with lexical or lexicon-based features consistently got the highest performance than the other five models, mainly on the structurally distinct genres such as Recipe and the Notice genre.

The best result was seen in the model that was using lexicon features with character-cleaned input, which suggets that to enhance genre separation, semantic grouping was a necessity. In contrast, Decision Tree classifier underpeformed when it was paired with sparse or syntactic features, showing that these models cannot do generaliation well to thenoisy or statistically diverse text.

By analyzing the misclassified pattern, it highlighted the challenges in distinguishing between two different genres, especially Letter and Review, which often do traits such as lexical and stylistic overlap with more formulaic genres like Notice. This gives the understanding of the complexity of genre boundaries in historical texts.

The limitations such as usage of default parameters, a relatively small feature set and there is a possibility that some misclassification is actually showing genuine genre hybridity and not model failure. Despite these limitations mentioned, the findings give the broader level conclusion that the surface-level lexical and semantic cues are the best effective indicators of genre in historical corpora. And that is by doing feature selection and preprocessing carefully, it can get and lead to meaningful performance improvement.

These particular outcomes give a basis for the future work in the historical genre modelling, which will include more granular genre definitions and binary genre pair analysis for richer text interpretations.
